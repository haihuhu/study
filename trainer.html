☢️ 绝对火力覆盖：9 大核心工程模块

TOP 1

1. Auth 登录权限系统

护城河的铁门。决定谁能进，谁能看，谁是管理员。

Clerk Middleware RBAC

基准 (D1-D5)：接入登录页。配置 Middleware，未登录强跳首页。

🔥 Day 7 变形目标：

  • 角色拦截：给用户分配 Admin 标签，普通用户访问 `/teacher` 报 401 拦截。
  • 数据隔离：查询时强制 `where: userId`，只能看自己的数据。

2. 企业级表单 + 验证

不要相信用户的任何输入，把垃圾数据挡在前端。

React Hook Form Zod shadcn/form

基准 (D1-D5)：构建包含文本、下拉框的表单，前端实时报红线（长度/邮箱验证）。

🔥 Day 7 变形目标：

  • 异步联动：选了 CategoryA，下方的下拉框动态刷新为其子分类。
  • UX 防抖：提交时按钮呈现 Spinner Loading 并禁用，成功后发 Toast。

3. API Route & Server Actions

系统的心脏血脉。处理内部数据交互与对外接口暴露。

Server Actions API Routes CORS

基准 (D1-D5)：用 Server Action 接收表单并打印；暴露纯粹的 `/api/test` GET 接口。

🔥 Day 7 变形目标：

  • 后端双重校验：在 Server Action 内部再跑一次 Zod，防恶意注入。
  • 为 API Route 加上 CORS 头和鉴权验证。

4. 终极 CRUD 与数据库

所有 SaaS 软件的底盘。增删改查决定了业务的厚度。

Prisma Schema Relations revalidatePath

基准 (D1-D5)：定义 Schema，实现基础的 `create`, `findMany`, `update`, `delete`。

🔥 Day 7 变形目标：

  • 一对多关联插入：创建 Course 的同时，自动在关联表创建一条 Chapter。
  • 软删除：不直接 Delete 数据，而是把字段 `isPublished` 设为 false。

5. Stripe 支付闭环

代码换美元的物理通道。掌握 Checkout 与异步回调。

Stripe Checkout Webhook API

基准 (D1-D5)：点击购买，唤起 Stripe 结账页，付完跳转回成功页。

🔥 Day 7 变形目标：

  • 实现“按月 Subscription 订阅模式”。
  • 在 API Webhook 中接收支付成功事件，操作 Prisma 把用户设为 VIP。

6. 文件与富媒体管理

现代应用刚需。处理图片直传、视频 Webhook 状态同步。

UploadThing Mux Video

基准 (D1-D5)：实现头像/封面上传，拿到远端 URL 存入库。

🔥 Day 7 变形目标：

  • 限制条件：只允许传 PDF，且小于 4MB。
  • 垃圾回收：点击删除课程时，调用远端 API 彻底删掉云存储中的旧图。

7. 复杂数据看板 (B端)

搞定企业级单子的利器。复杂表格、分页、图表可视化。

TanStack Table Recharts dnd-kit

基准 (D1-D5)：用 Data Table 渲染 Prisma 数据，支持列筛选。

🔥 Day 7 变形目标：

  • 服务端真分页：抛弃前端假分页，使用 Prisma 的 skip 和 take。
  • 拖拽排序：利用 dnd-kit 实现列表拖动，批量更新数据库 position 字段。

8. AI Agent 大模型集成

Upwork 高溢价风口。赋予你的系统理解自然语言的能力。

Vercel AI SDK Streaming OpenAI API

基准 (D1-D5)：用 `useChat` 实现打字机效果的 ChatGPT 对话框。

🔥 Day 7 变形目标：

  • System Prompt：强迫 AI 只能用“严厉的口吻”回答特定专业问题。
  • 结构化输出：用 `generateObject` 强制 AI 输出纯 JSON 格式数据。

🚀 9. 部署交付 (最后一公里)

代码没上线就是废纸。这是交付赚取评价的生死关。

Vercel Neon/Supabase Env

基准 (D1-D5)：申请云数据库 (Neon)，完成 `prisma db push`，Vercel 绑定 GitHub 自动部署。

🔥 Day 7 变形目标：

  • 彻底排查 TypeScript 打包黄线：在本地跑 `npm run build` 不报错才能 Push。
  • 线上变量管理：在 Vercel 准确配置线上专用的 Webhook Secret 和 API Key。
